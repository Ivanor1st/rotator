                v1.0 · Proxy intelligent
              

              
API Rotator

              

                Un proxy local compatible OpenAI qui route automatiquement vos requêtes vers le meilleur modèle disponible parmi vos providers configurés, avec rotation des clés, gestion des quotas et fallback automatique.
              

              

                
                
              

            


            

              

                
🔀

                
Rotation automatique

                
Distribue les requêtes entre vos clés pour éviter les limites de débit.

              

              

                
🧠

                
Routage par profil

                
7 profils prédéfinis (coding, reasoning, chat…) avec sélection du meilleur modèle.

              

              

                
🔄

                
Fallback automatique

                
Si un provider échoue, le suivant est utilisé transparemment.

              

              

                
💻

                
Ollama local

                
Fallback infini sur vos modèles locaux. Aucun quota, disponible hors ligne.

              

              

                
🔑

                
Multi-clés

                
Ajoutez plusieurs clés par provider pour multiplier vos quotas gratuits.

              

              

                
📊

                
Dashboard temps réel

                
Monitoring des quotas, logs d'activité et statistiques détaillées.

⚡ Démarrage rapide

            
Configurez et utilisez votre proxy en moins de 2 minutes.


            

              ℹ️
              
Le proxy démarre automatiquement. Même sans clé configurée, les modèles Ollama locaux sont disponibles comme fallback.

            


            

              
Étapes de configuration

              

                

                  
1

                  

                    
Copier l'URL de base

                    
Utilisez http://localhost:47822/v1 comme base URL dans votre application. La clé API est rotator.

                  

                

                

                  
2

                  

                    
Ajouter vos clés API

                    
Rendez-vous dans Mes clés API et ajoutez vos clés pour chaque provider. Testez-les avant de sauvegarder pour valider qu'elles fonctionnent.

                  

                

                

                  
3

                  

                    
Choisir votre modèle

                    
Utilisez un nom de profil (coding, chat, reasoning…) comme modèle dans votre application. Le proxy sélectionne automatiquement le meilleur disponible.

                  

                

                

                  
4

                  

                    
Vérifier l'état

                    
La Vue d'ensemble affiche les providers actifs, quotas consommés et dernières requêtes en temps réel.

                  

                

              

            


            

              

                Python (openai SDK)
                
              

              
from openai import OpenAI

client = OpenAI(
    base_url="http://localhost:47822/v1",
    api_key="rotator"
)

response = client.chat.completions.create(
    model="coding",   # ou "chat", "reasoning", "vision"...
    messages=[{"role": "user", "content": "Hello!"}]
)
print(response.choices[0].message.content)

🔀 Providers

            
Chaque provider est un fournisseur de modèles avec ses propres clés et quotas.


            

              ⚠️
              
La rotation des clés est automatique. Chaque clé est utilisée à tour de rôle pour maximiser les quotas disponibles. Si une clé est épuisée, la suivante est utilisée sans interruption.

            


            

              
Comportement de rotation

              

                

                  
✓

                  

                    
Ordre de priorité

                    
Par défaut : Google → NVIDIA → OpenRouter → Ollama Cloud → Local. Modifiable via les presets.

                  

                

                

                  
✓

                  

                    
Suspension automatique

                    
Un provider qui retourne des erreurs 429 (quota) est automatiquement suspendu et exclu du routage pendant 1h.

                  

                

                

                  
✓

                  

                    
Réactivation automatique

                    
Un provider suspendu est automatiquement retesté et réactivé après la période de suspension.

                  

                

              

            


            

              
IdentifiantRPM maxClés multiplesNotes

Stratégies pour maximiser vos quotas gratuits en multipliant les clés.


            

              🔐
              
Les clés sont stockées en clair dans config.yaml. Ne commitez jamais ce fichier dans un dépôt public et ne partagez pas de captures d'écran de la section Configuration.

            


            

              
Maximiser les quotas gratuits

              

                

                  
1

                  

                    
Plusieurs comptes Google

                    
Créez plusieurs comptes Google et générez une clé Gemini API pour chacun. Chaque clé offre 14 400 requêtes/jour. 3 clés = 43 200 req/jour.

                  

                

                

                  
2

                  

                    
Multi-clés NVIDIA

                    
NVIDIA offre un crédit gratuit généreux à chaque nouveau compte. Créez plusieurs comptes avec différentes adresses email.

                  

                

                

                  
3

                  

                    
Tester avant de sauvegarder

                    
Utilisez le bouton 🧪 sur chaque clé avant de sauvegarder. Les clés invalides sont refusées pour éviter des erreurs en production.

                  

                

              

            

          


          
          

            
🧷 Projets & tokens

            
Isolez les quotas et les logs par application ou utilisateur via des tokens de projet.


            

              
Fonctionnement

              

                

                  
→

                  

                    
Un token par projet

                    
Chaque projet reçoit un token unique. Utilisez ce token comme clé API dans votre application au lieu de rotator.

                  

                

                

                  
→

                  

                    
Politiques d'accès

                    
full_access : accès complet. coding_only : uniquement le profil coding (idéal pour Claude Code).

                  

                

                

                  
→

                  

                    
Modes de quota

                    
hard_block : erreur 429 au-delà du quota. local_only : bascule sur Ollama local. alert_only : continue mais alerte.

🔐 Sécurité

            
Bonnes pratiques pour utiliser API Rotator en toute sécurité.


            

              🚨
              
Règle absolue : Ne versionnez jamais config.yaml avec des clés réelles. Ajoutez-le à votre .gitignore.

            


            

              
Liste de contrôle sécurité

              

                
✓
config.yaml dans .gitignore
Ajoutez config.yaml et *.yaml à votre .gitignore.

                
✓
Proxy local uniquement
N'exposez jamais le port 47822 sur internet. Il est conçu pour un usage local uniquement.

                
✓
Tokens de projet pour partage
Si vous partagez l'accès, créez un token de projet avec une politique restrictive plutôt que de partager la clé rotator.

                
✓
Révoquer les clés compromises
En cas de fuite, révoquez immédiatement la clé chez le provider (Google AI Studio, NVIDIA…) puis mettez-la à jour dans le proxy.

Solutions aux problèmes les plus courants.


            

              
Erreurs fréquentes

              

                
SymptômeCause probableSolution

                
                  

                    Connection refused
                    Le proxy ne tourne pas
                    Vérifiez que le processus est lancé. Redémarrez-le.
                  

                  

                    401 Unauthorized
                    Clé API mal configurée
                    Testez la clé dans l'onglet Tests & Benchmark.
                  

                  

                    429 Too Many Requests
                    Quota épuisé
                    Ajoutez plus de clés ou attendez le reset (minuit UTC pour Google).
                  

                  

                    Modèle non trouvé

                    Consultez /v1/models pour la liste exacte.
                  

                  

                    Ollama non détecté
                    Ollama non installé/démarré
                    Installez Ollama depuis la Vue d'ensemble ou démarrez-le manuellement.
