🦞 Qu'est-ce qu'OpenClaw ?

            

              OpenClaw est une passerelle auto-hébergée qui connecte vos apps de
              messagerie
              (WhatsApp, Telegram, Discord, iMessage, Slack, Signal, Mattermost, Google Chat…) à des agents IA.


              En le connectant au Rotator, vous obtenez un assistant IA personnel
              accessible depuis n'importe quelle app de messagerie,
              routé automatiquement vers le meilleur modèle disponible parmi tous vos providers.


              Le dashboard ci-dessus vous donne un contrôle total sur votre installation OpenClaw :
              agents, skills,
              channels, cron jobs,
              sessions, tools,
              et diagnostics.
            

          


          

            
🚀 Démarrage rapide

            

              
# 1. Installer OpenClaw

              
npm install -g openclaw@latest

              
# 2. Lancer le wizard d'onboarding

              
openclaw onboard --install-daemon

              
# 3. Connecter WhatsApp

              
openclaw channels login

              
# 4. Démarrer le gateway

              
openclaw gateway --port 18789

              
# 5. Lancer un diagnostic

              
openclaw doctor

            

          


          

            
🔧 Connecter au Rotator

            

              Utilisez le bouton "Configurer Rotator comme provider" dans l'onglet Connexion
              (ou Config) pour injecter automatiquement le provider rotator dans
              votre openclaw.json.
              Cela crée 4 modèles : coding, reasoning, chat, long.

                  v1.0 · Proxy intelligent
                

                
API Rotator

                

                  Un proxy local compatible OpenAI qui route automatiquement vos requêtes vers le meilleur modèle
                  disponible parmi vos providers configurés, avec rotation des clés, gestion des quotas et fallback
                  automatique.
                

                

                  
                  
                

              


              

                

                  
🔀

                  
Rotation automatique

                  
Distribue les requêtes entre vos clés pour éviter les limites de débit.
                  

                

                

                  
🧠

                  
Routage par profil

                  
7 profils prédéfinis (coding, reasoning, chat…) avec sélection du
                    meilleur modèle.

                

                

                  
🔄

                  
Fallback automatique

                  
Si un provider échoue, le suivant est utilisé transparemment.

                

                

                  
💻

                  
Ollama local

                  
Fallback infini sur vos modèles locaux. Aucun quota, disponible hors
                    ligne.

                

                

                  
🔑

                  
Multi-clés

                  
Ajoutez plusieurs clés par provider pour multiplier vos quotas gratuits.
                  

                

                

                  
📊

                  
Dashboard temps réel

                  
Monitoring des quotas, logs d'activité et statistiques détaillées.

🔀 Providers

              
Chaque provider est un fournisseur de modèles avec ses propres clés et
                quotas.


              

                ⚠️
                
La rotation des clés est automatique. Chaque clé est utilisée à tour de rôle pour maximiser les
                  quotas disponibles. Si une clé est épuisée, la suivante est utilisée sans interruption.

              


              

                
Comportement de rotation

                

                  

                    
✓

                    

                      
Ordre de priorité

                      
Par défaut : Google → NVIDIA → OpenRouter → Ollama Cloud → Local.
                        Modifiable via les presets.

                    

                  

                  

                    
✓

                    

                      
Suspension automatique

                      
Un provider qui retourne des erreurs 429 (quota) est automatiquement
                        suspendu et exclu du routage pendant 1h.

                    

                  

                  

                    
✓

                    

                      
Réactivation automatique

                      
Un provider suspendu est automatiquement retesté et réactivé après la
                        période de suspension.

Maximiser les quotas gratuits

                

                  

                    
1

                    

                      
Plusieurs comptes Google

                      
Créez plusieurs comptes Google et générez une clé Gemini API pour
                        chacun. Chaque clé offre 14 400 requêtes/jour. 3 clés = 43 200 req/jour.

                    

                  

                  

                    
2

                    

                      
Multi-clés NVIDIA

                      
NVIDIA offre un crédit gratuit généreux à chaque nouveau compte. Créez
                        plusieurs comptes avec différentes adresses email.

                    

                  

                  

                    
3

                    

                      
Tester avant de sauvegarder

                      
Utilisez le bouton 🧪 sur chaque clé avant de sauvegarder. Les clés
                        invalides sont refusées pour éviter des erreurs en production.

Politiques d'accès

                      
full_access
                        : accès complet. coding_only
                        : uniquement le profil coding (idéal pour Claude Code).

                    

                  

                  

                    
→

                    

                      
Modes de quota

                      
hard_block
                        : erreur 429 au-delà du quota. local_only
                        : bascule sur Ollama local. alert_only
                        : continue mais alerte.

                    

                  

                

              

            


            
            

              
🕐 Planification

              
Automatisez des changements de comportement à des horaires précis.


              

                
Actions disponibles

                

                  
                    

                      Action
                      Cible
                      Effet
                    

                  
                  
                    

                      block
                      provider_id
                      Suspend un provider pendant la plage horaire
                    

                    

                      force
                      profile:provider
                      Force un profil vers un provider spécifique
                    

                    

                      switch_preset
                      preset_name
                      Active un preset pendant la plage horaire
                    

                  
                

              


              

                
Exemple — mode nuit économique
                

                
Nom    : Nuit économique
Action : block
Cible  : nvidia
Début  : 22:00
Fin    : 07:00
Jours  : all

              

            


            
            

              
🔐 Sécurité

              
Bonnes pratiques pour utiliser API Rotator en toute sécurité.


              

                🚨
                
Règle absolue : Ne versionnez jamais config.yaml avec des clés réelles.
                  Ajoutez-le à votre .gitignore.

              


              

                
Liste de contrôle sécurité

                

                  

                    
✓

                    

                      
config.yaml dans .gitignore

                      
Ajoutez config.yaml
                        et *.yaml
                        à votre .gitignore.

                    

                  

                  

                    
✓

                    

                      
Proxy local uniquement

                      
N'exposez jamais le port 47822 sur internet. Il est conçu pour un usage
                        local uniquement.

                    

                  

                  

                    
✓

                    

                      
Tokens de projet pour partage

                      
Si vous partagez l'accès, créez un token de projet avec une politique
                        restrictive plutôt que de partager la clé rotator.
                      

                    

                  

                  

                    
✓

                    

                      
Révoquer les clés compromises

                      
En cas de fuite, révoquez immédiatement la clé chez le provider (Google
                        AI Studio, NVIDIA…) puis mettez-la à jour dans le proxy.

                    

                  

                

              

            


            
            

              
🛠 Dépannage

              
Solutions aux problèmes les plus courants.


              

                
Erreurs fréquentes

                

                  
                    

                      Symptôme
                      Cause probable
                      Solution
                    

                  
                  
                    

                      Connection refused
                      Le proxy ne tourne pas
                      Vérifiez que le processus est lancé. Redémarrez-le.
                    

                    

                      401 Unauthorized
                      Clé API mal configurée
                      Testez la clé dans l'onglet Tests & Benchmark.
                    

                    

                      429 Too Many Requests
                      Quota épuisé
                      Ajoutez plus de clés ou attendez le reset (minuit UTC pour Google).
                    

                    

                      Modèle non trouvé
                      Nom de profil incorrect
                      Consultez /v1/models pour la liste exacte.
                      
                    

                    

                      Ollama non détecté
                      Ollama non installé/démarré
                      Installez Ollama depuis la Vue d'ensemble ou démarrez-le manuellement.
